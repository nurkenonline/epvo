# RF_TFW-1.14 — Системные справочники (Center / System Dictionaries)

> **Группа:** Централизованные (неизменяемые вузом) справочники ЕПВО
> **Сущностей:** 50+ | **Endpoint:** `GET /api/v1/system-dictionary/{dictionaryCode}`
> **Формат:** каждый справочник — массив `{ id, nameRu, nameKz, nameEn }` (+ `code` у некоторых)

---

## Общая структура системных справочников

Все системные справочники имеют **одинаковую** структуру:

```json
{
  "id": 1,
  "nameRu": "Название на русском",
  "nameKz": "Название на казахском",
  "nameEn": "Name in English"
}
```

Некоторые справочники (например, `CenterProfession`) имеют расширенные поля: `code`, `parentId`, `levelId` и т.д.

## Примечания

1. **Системные справочники НЕ редактируются вузом** — они управляются централизованно из ЕПВО.
2. **При передаче org-data** значения `int32` (FK) ссылаются на `id` из соответствующего системного справочника.
3. **Для получения актуальных данных** используйте эндпоинт:
   ```
   GET /api/v1/system-dictionary/{dictionaryCode}
   ```
4. Набор справочников может расширяться — см. enum `SystemDictionaryCode` в OpenAPI spec.
