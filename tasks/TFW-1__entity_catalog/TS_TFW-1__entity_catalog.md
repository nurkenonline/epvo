# TS_TFW-1 — Задача: описать JSON-схемы всех сущностей для передачи в ЕПВО

## Цель
Для каждой из 90 сущностей `OrgDataDictionaryCode` описать полную JSON-структуру, которую необходимо передать через `POST /org-data/list/save`. Результат — набор RF-файлов с JSON-примерами + описанием каждого поля.

## Контекст
- HL-файл: `HL_TFW-1__entity_catalog.md` — содержит полный каталог сущностей, их группировку и маппинг составных ключей.
- Источник API: OpenAPI spec в `RF__api_spec.json`

## Границы задачи

### In Scope
- Для каждой сущности: `typeCode`, полный набор полей с типами данных, описаниями, обязательностью
- JSON-примеры (Example Value) для каждой сущности
- Маппинг `type` (составной ключ) для каждой сущности
- Описание ссылочных полей (FK): какие ID ссылаются на какие справочники
- Группировка сущностей по логическим блокам (как в HL)

### Out of Scope
- Откуда берутся данные (источник — задача другого агента)
- Реализация кода API-клиента (TFW-2)
- Порядок загрузки (TFW-3)

## Definition of Done

1. ✅ Для каждой из 90 сущностей есть описание:
   - Название и `typeCode`
   - `type` (составной ключ) для find-by-id / delete
   - Таблица полей: имя, тип, обязательность, описание, FK-связь
   - JSON-пример объекта для `POST /org-data/list/save`
2. ✅ Для каждого системного справочника: имя, код, набор полей
3. ✅ Все FK-ссылки между сущностями задокументированы
4. ✅ RF-файлы с примерами JSON для каждой группы сущностей

## Формат RF-файлов

Для каждой сущности в RF-файле:

```markdown
## ENTITY_CODE — Описание

- **typeCode**: `ENTITY_CODE`
- **Composite Key type**: `XXX_ID_COMPOSITE_KEY`
- **Composite Key поле**: `entityId` (int32)
- **Зависимости**: ссылается на FACULTIES (facultyId), TUTOR (tutorId)

### Поля

| Поле | Тип | Обязательное | Описание | FK-ссылка |
|------|-----|-------------|----------|-----------|
| typeCode | string | ✅ | Дискриминатор | OrgDataDictionaryCode |
| universityId | int32 | ✅ | ID вуза | - |
| entityId | int32 | ✅ | Уникальный ID | PK |
| nameRu | string | ❌ | Название (RU) | - |
| nameKz | string | ❌ | Название (KZ) | - |
| facultyId | int32 | ❌ | ID факультета | → FACULTIES |

### JSON-пример
​```json
{
  "typeCode": "ENTITY_CODE",
  "universityId": 12345,
  "entityId": 1,
  "nameRu": "...",
  "nameKz": "...",
  "facultyId": 10
}
​```
```

## Инструкции для ИИ-агента

1. Откройте `HL_TFW-1__entity_catalog.md` для понимания группировки
2. Загрузите OpenAPI spec из `RF__api_spec.json` (или `https://epvo.kz/isvuz/api/v3/api-docs`)
3. Для каждой сущности в вашей группе:
   a. Найдите схему в `components.schemas`
   b. Извлеките все поля, типы, описания, required-поля
   c. Определите FK-ссылки по смыслу полей (напр. `facultyId` → FACULTIES)
   d. Создайте JSON-пример с реалистичными данными
4. Результат сохраните как `RF_TFW-1.N__group_name.md`
