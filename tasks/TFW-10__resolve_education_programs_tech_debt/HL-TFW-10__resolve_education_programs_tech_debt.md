# HL — TFW-10: Разрешение технического долга по Реестру ОП

> **Автор**: Coordinator (AI)
> **Статус**: ✅ HL — Выполнено

## 1. Видение
В рамках TFW-9 был задокументирован правильный процесс синхронизации данных образовательных программ (ОП) с ЕПВО. Однако в нашем Entity Catalog (`RF_TFW-1.2`) и Gap Analysis (`RF_TFW-4.B`) сохранились критические неточности, которые могут сбить с толку разработчиков на этапе имплементации. Задача TFW-10 (Resolution) заключается в очистке документации и закрытии зафиксированных долгов (TD-21 — TD-25).

## 2. Текущее состояние (As-Is)
- `RF_TFW-1.2`: описывает `SPECIALIZATIONS` как сущность из 10 полей с прямой ссылкой (FK) на `professionId`. Реальное API требует до 30 полей и связывает ОП с ГОП через мостовую таблицу (`prof_caf_id` → `PROFESSION_CAFEDRA`).
- `RF_TFW-4.B`: использует устаревшее название сущности (`EducationProgram` вместо `SPECIALIZATIONS`) и единицы измерения `duration` (в месяцах), в то время как `studyPeriod` для ГОП измеряется в семестрах.
- `KNOWLEDGE.md`: не содержит базовых архитектурных принципов (Knowledge Items) о механизме загрузки базы ОП.

## 3. Целевое состояние (To-Be)
1. **Единая версия правды:** `RF_TFW-1.2` приведён в соответствие с требованиями `adm_doc` (добавлены пропущенные поля, исправлены ключи).
2. **Семантическая ясность:** В `RF_TFW-4.B` проставлены предупреждающие аннотации (deprecation warnings / clarifications).
3. **Обновлённая База знаний:** Новая секция в `KNOWLEDGE.md` фиксирует базовые правила передачи ОП.

## 4. Scope
Задача выполняется в одну фазу. Изменения затрагивают только существующую документацию (RF-артефакты и реестры). Написание нового программного кода не требуется.

### Затрагиваемые артефакты:
- 🟢 `RF_TFW-1.2__education_programs.md` (добавление полей и исправление графа зависимостей)
- 🟢 `RF_TFW-4.B__admissions_registries.md` (добавление Alert-блоков для разработчиков)
- 🟢 `KNOWLEDGE.md` (добавление новой секции §2.12 или §3.x)
- 🟢 `TECH_DEBT.md` (отметка TD-21 – TD-25 как решённых)

## 5. Definition of Done (DoD)
- ✅ 1. **TD-21, TD-22:** В `RF_TFW-1.2` секция `SPECIALIZATIONS` дополнена (теперь 30+ полей), поле `professionId` заменено на `prof_caf_id`, обновлён Mermaid-граф. Указана ссылка на TFW-10.
- ✅ 2. **TD-23, TD-25:** В `RF_TFW-4.B` добавлено `> [!WARNING]` об отличиях `duration` от `studyPeriod` и уточнено, что API name = `SPECIALIZATIONS`.
- ✅ 3. **TD-24:** В `KNOWLEDGE.md` добавлен раздел о Реестре Образовательных Программ с выдержкой архитектурных правил (например, обязательность `center_profession_id` и `prof_caf_id`).
- ✅ 4. Долги TD-21 — TD-25 в `TECH_DEBT.md` переведены в статус `✅ Resolved (TFW-10)`.

## 6. Принципы
- **Safe Modifications:** Не удалять существующие полезные данные, только аннотировать и расширять.
- **Traceability:** Добавлять ссылки «*Обновлено в TFW-10*» в модифицированных файлах.
