## 1. Видение
Четвертая фаза декомпозиции (Phase D) отвечает за модуль "Финансирование". Этот модуль критически важен для выплаты государственных стипендий. ЕПВО строго контролирует начисление средств, опираясь на академические успехи (оценки из Phase C), банковские реквизиты (ИИК/БИК) и социальный статус студента (сироты, гос. обеспечение). Наша задача — описать бизнес-правила (логику), по которым ВУЗ должен фильтровать и передавать эти данные в ЕПВО.

## 2. Область применения (Scope)
- **Целевые сущности ЕПВО:** 
  - `Student` (расширение профиля: поля `iic`, `bic` для банковских счетов).
  - `Scholarship` (Назначенная стипендия, сумма, дата выплаты).
  - `StudentBenefit` (Льготы: `fullProvision` - полное гос. обеспечение).
- **Ключевые блоки бизнес-логики:**
  - Логика расчета `overall_performance` (показатель "стипендиальной" успеваемости) на основе транскриптов.
  - Условия передачи банковских реквизитов (только для грантников или для всех?).
  - Логика привязки стипендии к приказу (Order type = 13).
- **Бизнес-цикл:** Ежемесячно (обычно до 10 числа каждого месяца для формирования выплат).

## 3. Целевое состояние (To-Be)
В результате фазы D должен быть создан завизированный артефакт `RF_TFW-4.D`, который будет содержать:

2. Псевдокод (Business rules) по определению права студента на стипендию (отсутствие оценок ниже "B-", "C" и т.д., согласно правилам).
3. Инструкции по связыванию `Scholarship` с `OrderSection`.
