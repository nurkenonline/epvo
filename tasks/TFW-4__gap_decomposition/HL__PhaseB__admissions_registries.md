## 1. Видение
Вторая фаза декомпозиции (Phase B) фокусируется на статичных справочниках ВУЗа, которые редко меняются во времени, но критично важны для привязки всех остальных данных (оценок, приказов, дипломов). Наша цель — формализовать правила сопоставления внутренних образовательных программ (ОП), квалификаций и специальностей ВУЗа с едиными системными классификаторами ЕПВО.

## 2. Область применения (Scope)
- **Целевые сущности ЕПВО:** 
  - `EducationProgram` (Образовательные программы).
  - Маппинг `Profession` (Специальности) через `CenterProfession`.
  - Маппинг справочников (Формы обучения, Языки обучения, Степени).
- **Ключевые блоки бизнес-логики:**
  - Логика поиска и сопоставления локального ID специальности с системным кодом (например, ОКЭД/МСКО).
  - Обязательные атрибуты ОП (Срок обучения, кол-во кредитов).
- **Бизнес-цикл:** Раз в год (перед началом учебного года или при лицензировании новых ОП).

## 3. Целевое состояние (To-Be)
В результате фазы B должен быть создан завизированный артефакт `RF_TFW-4.B`, который будет содержать:
1. Матрицы полей для справочников и ОП.
2. Алгоритмы маппинга локальных ID на системные ID ЕПВО (псевдокод).
3. Обработка сценариев, когда локальной специальности нет в системном справочнике.
