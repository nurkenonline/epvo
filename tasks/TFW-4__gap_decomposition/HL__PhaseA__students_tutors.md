## 1. Видение
Первая фаза декомпозиции (Phase A) фокусируется на фундаментальном ядре данных ЕПВО: физические лица (обучающиеся и преподаватели). Наша цель — формализовать строгие бизнес-правила и условия трансформации данных ВУЗа в требуемый формат ЕПВО для работы со статусами ОСМС, гражданством и базовой категоризацией. Этот документ станет спецификацией для написания мапперов (ETL-процессов).

## 2. Область применения (Scope)
- **Целевые сущности ЕПВО:** 
  - `STUDENT` (Студенты бакалавриата, магистратуры, докторантуры)
  - `STUDENT_INFO` (Расширенные данные студентов)
  - `TUTOR` (Профессорско-преподавательский состав)
- **Ключевые блоки бизнес-логики:**
  - Персональные данные (ИИН, пол, валидация 7-го разряда).
  - Гражданство и паспорта (`citizenshipId`).
  - Адресные данные (КАТО).
  - Статусы ОСМС (Medical Insurance) и флаг `isstudent`.
- **Бизнес-цикл:** Ежедневное обновление (Data Sync).

## 3. Целевое состояние (To-Be)
В результате фазы A должен быть создан завизированный артефакт `RF_TFW-4.A`, который будет содержать:
1. Матрицу полей указанных сущностей.
2. Правила валидации **на естественном языке с элементами псевдокода** (например, "Если `citizenshipId != 113`, то `incorrectiin = true`").
3. Источники данных в базе AD (Theoretical Source).
4. Ограничения и известные ошибки API (из `KNOWLEDGE.md`).

## 4. Ограничения бюджета (Scope Budget)
Согласно правилам координатора `tfw-plan.md` для одной фазы:
- Файлов в фазе: 2-3 (HL, TS, финальный RF).
- Фокус строго на контингенте (без оценок, финансов или приказов). Приказы вынесены в Phase C.
