## 1. Спецификация (Specification)
Этот документ описывает технические требования к разработке итогового мануала `RF_TFW-4.A__students_tutors.md`.
Артефакт должен быть написан так, чтобы любой ИИ-агент, реализующий интеграцию на стороне кода (Python/Java), мог взять его как единственную спецификацию (Source of Truth) для написания DTO и Mapper-классов.

### 1.1. Ожидаемая структура RF
Итоговый RF документ (`RF_TFW-4.A`) должен строго следовать этому шаблону:
1. **Сущность (например, STUDENT):**
   - **Endpoint:** `/org-data/list/save`
   - **Composite Key:** Признак уникальности (например, `studentId`).
2. **Матрица маппинга:**
   Таблица с колонками: `EPVO Field` | `Type` | `Required` | `AD Source / Logic` | `Default Validations`.
3. **Блок жесткой бизнес-логики (Critical Rules):**
   - Условия ветвления (псевдокод).
   - Привязки к `KNOWLEDGE.md`.

### 1.2. Базовые правила маппинга для Phase A
Executor должен извлечь и формализовать в RF:
- **Гражданство:** Обязательность `citizenshipId`. Если локально NULL, то подставлять ID страны по умолчанию (113 - Казахстан), либо фильтровать запись.
- **ИИН и Пол:** Описать логику обхода (override) валидации 7-го разряда.
  - `IF citizenshipId != 113 THEN override = false`
  - `IF (citizenshipId == 113) AND (IIN 7th digit wrong) THEN incorrectiin = true`.
- **ОСМС (Мед. страховка):** Описать правила формирования статуса `isstudent` и условия его валидности (дополнительные поля при статусах отчислен/обучается).
- **КАТО:** Описать трансформацию полного региона КАТО (Астана) в ID справочника КАТО ЕПВО.

## 2. Качественный контракт (Quality Contract)
| Правило | Обоснование |
|---------|-------------|
| Не использовать "просто текстовые" описания маппинга. Использовать псевдокод (`IF/ELSE` или `switch`). | Реверс-инжиниринг кода требует точных и однозначных алгоритмов программирования. |
| Ссылаться на ID системных справочников `Center...` при упоминании FK. | ВУЗ должен знать, к каким таблицам ЕПВО идет привязка (Например: `CenterCountry`). |
| Игнорировать удаленные поля. | Если в `RF_TFW-2.3` поле помечено как deprecated, не прописывать маппинг. |
