## 1. Цель

Создать единый RF-документ `RF__TFW-5__student_orders.md`, систематизирующий все 25+ типов приказов ЕПВО (`orderType`) в формате справочника с pipeline-описаниями, побочными эффектами на STUDENT, правилами `movement_date` и маппингом `retire_reason`.

## 2. Scope

### In Scope
- Полный реестр 25+ `orderType` с категориями (`categoryId`) и пометками `is_show=0`
- Pipeline каждого контингентного типа (порядок отправки сущностей)
- Побочные эффекты на профиль STUDENT (`isStudent`, `isinretire`, `courseNumber`, `academicMobility`)
- Правила обязательности `movement_date` по типам/категориям
- Маппинг `retire_reason` (значения 1–9) для type=6/14
- Правила `ordersadditional` (какие поля для каких типов/категорий)
- Таблица пробелов (orderType=40, деактивированные категории)
- Mermaid-диаграмма иерархии сущностей приказов

### Out of Scope
- Написание кода интеграции
- Практическое тестирование API
- Документирование сущностей, не связанных с приказами

## 4. Детальные шаги

### Step 1: Структура RF-документа

Создать RF с секциями:

3. Иерархия сущностей — mermaid-диаграмма (из HL §2.4)

### Step 2: Pipeline-описания для контингентных типов

Для каждого из 18 контингентных orderType (2, 3, 4, 6, 7, 8, 9, 10, 11, 13, 14, 27, 28, 30, 31, 32, 33, 34) описать:

**Источники:**

- `RF_TFW-4.D` (стипендии)
- `RF_TFW-4.E` (выпускники)
